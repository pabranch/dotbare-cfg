Personal persistent context for coding agents working with Perry. Loaded via reference from, for example from `~/.pi/agent/AGENTS.md`. Project and Agent agnostic — applies to every session unless overridden by a project-local `CLAUDE.md` or `AGENTS.md` file.

## Git workflow

- `autosquash = true` is set globally in `~/.gitconfig`. Combined with
  `git rbi` (alias for `rebase --interactive --autosquash`) this
  automatically folds `git fixup` commits into their target.
- Use `git fixup <sha>` (`git commit --fixup=<sha>`) to mark a follow-up
  intended for an earlier commit. Use `git squash <sha>` for
  non-fast-forward folds.
- Stage discipline: stage related changes together for one logical
  commit; use fixup commits for edits that belong on a prior commit;
  leave the working tree clean after each commit. Don't bundle
  unrelated fixes into the same commit.
- Other commonly-used aliases worth honoring:
  - `git lg` / `git ll`: compact / detailed log graphs
  - `git lsu`: `log -100000 @{u}^..` (everything ahead of upstream)
  - `git df` / `git dfc`: `diff -M -w` / `--cached`
  - `git st`: `status -sb` (short + branch info)
  - `git amend`: `commit --amend --no-edit`
  - `git fp`: `push --force-with-lease` (safe force-push)
- Perry is not afraid of rebase nor force pushing when interacting with repos he
  controls.

## Working style

- **Never push unless explicitly told to do so.** Even `git fp` and other
  safe-push forms stay hands-off until Perry asks.
- **Ask when uncertain.** Perry does not mind answering questions. If you
  are vacillating between two reasonable approaches, stop and ask rather
  than guessing. Clarification is cheaper than a wrong move.
- **Append freely to this file.** You are permitted to update `~/.PERRY.md`
  whenever you notice something that would streamline future work together.
  Add new sections; do not silently rewrite existing content.
- **Lean context.** Concise summaries and tables are appreciated over
  running prose. Context tokens = cost, so prefer tight context files and
  link out for detail rather than duplicating.
